## 2025-02-28 - Bet Modal Accessibility and Polish
**Learning:** Dialog components often have decorative emojis in their titles that screen readers parse awkwardly ("smiling face with horns, Place Bet"). Also, disabled interactive states (like a submitting button) need clear visual indicators (like a spinner) so users aren't left wondering if their click registered, and custom buttons outside the standard library components often miss critical focus indicators for keyboard users.
**Action:** Always add `aria-hidden="true"` to purely decorative emojis. Ensure disabled states during async actions always include a visual loading indicator. Make sure all custom interactive elements (like quick-select pills) have `focus-visible` styles matching the design system (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`).## 2023-10-27 - Main Navigation and Header Accessibility
**Learning:** Decorative emojis in navigation components (like BottomNav) and header bars (like TopHeader/CoinBubble) will be awkwardly read out by screen readers unless hidden. Additionally, custom icon-only action buttons (share, theme toggle, logout) lack standard accessible names (`aria-label`) and keyboard focus outlines.
**Action:** Consistently apply `aria-hidden="true"` to decorative emojis across layout/navigation elements. Always add descriptive `aria-label`s to icon-only buttons. Always append standard Shadcn/Tailwind focus classes (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`) to all interactive elements lacking them. Use `aria-current="page"` for active navigation states.
